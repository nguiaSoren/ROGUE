## The one rule that governs everything

**The pool is plumbing; the assurance is the product.** ROGUE never builds or sells "faster coding," retrieval quality, or authoring UX (spec Â§1, build 08 Â§0). ## The surface-as-instrument map

Surface 3 is the *same instrument as Surfaces 1/2, pointed at accumulated knowledge*: take a target (the shared pool), fire inputs at it, score the result against an independent standard, emit a signed record. The mapping onto the shared instrument spine:

- **Target** = the **shared skill pool** â€” org/cohort-scoped `Skill` rows (`skill_md` + a pgvector `embedding`), the accumulated knowledge an agent fleet would draw on. The pool is "just another target" plugged into the scan engine's step-1 seam (ADR-0004 / unified Â§1), not a special case.
- **Probe** = an extraction/exfiltration attack (leakage), a held-out rollout pair (net-effect), or a co-injected top-k skill SET (combination) â€” depending on which number is being measured.
- **Consummation / breach** = the *produced* bad outcome, never the appearance of one: **recovery** of a protected datum (leakage), a **net-negative outcome** with vs without a skill (degradation), or a **harmful composition actually produced** by a co-invoked set. "Looks risky" / "could theoretically combine" / "mentions a generic pattern" is **engagement, not a breach**.
- **Ground truth** = a **canary** (leakage, canary-based recovery), the **net-effect judge** verdict on a held-out pair (degradation), and the **harmful-composition judge** verdict (combination) â€” each calibrated against an independently-labeled set with CIs before any number is claimed (ADR-0011).

## The four assured numbers (the signed Â§3 payload)

The exit-gate attestation carries exactly four pool numbers, each read back from the `skill_verifications` audit rows (never recomputed) and carrying provenance to a calibrated judge:

1. **N skills active, all verified net-positive with CIs** â€” the count from `kind=promotion` PASS rows + each skill's net-effect and bootstrap CI.
2. **measured leakage rate X% under extraction** â€” the latest `kind=leakage` rate + CI + the benchmark pack id + the coverage tier (the rate is only as strong as the pack; never quoted bare).
3. **M dangerous co-invocation neighborhoods quarantined** â€” the count from `kind=combination` PASS rows + the per-row risk/scan provenance.
4. **cohort isolation: no skill crossed a trust boundary it wasn't scoped to** â€” a summary assertion over the cohort scoping.
