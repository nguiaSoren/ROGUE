## ✅ #TRS-C — Grammar-component predictive-power (observational, $0) — RAN 2026-06-06: VERDICT weak/none

**⚑ The finding (possibly publishable — a clean negative result).**

**VERDICT: weak/none — and that's the high-value outcome here.** Run live against 351 primitives / 301 with breach data / 1,540 (primitive × target) units:

- **Marginal lift exists but is family-driven, not grammar-driven.** The only strong movers — `multimodal` (OR 4.56), `training_data_extraction` (OR 3.35) — are family-mirroring nodes, flagged circular by the Cramér's-V collinearity check (the node ≈ the family label). This directly tests the memo's own hypothesis ("RoleHijack + AuthorityFrame = 2.5× breach odds"): in current data, `role_hijack` node OR = 0.95 (no lift), and the composition did not survive controls. So the specific premise the AST roadmap rests on is not supported by the existing corpus — exactly the cheap falsification you wanted before building the grammar engine. *Design record (the study as originally specified, now executed):*

**Status: DONE (verdict weak/none) — a $0 observational screen over existing primitives × breaches that can FALSIFY the grammar/AST roadmap cheaply or point the powered interventional test (#TRS-A) at the live hypotheses.** The Technique-AST and synthetic-generation roadmap rests on the assumption that grammar structure predicts breach outcomes — that nodes and node combinations in the `GrammarNode` representation carry statistically significant signal beyond attack-family membership. This study labels the existing corpus with structural grammar nodes and tests that assumption with proper controls over the existing telemetry, spending $0, running no generation, touching nothing in production.

**Research framing.** This is the cheap observational counterpart to the interventional powered experiment (#TRS-A): (a) **#TRS-C (this study, observational/correlational)** — tests whether grammar nodes and pairwise combinations carry predictive lift over the existing primitives × breaches corpus, with explicit confound controls. It is fast, free, and confounded — it can *falsify* the grammar hypothesis cheaply (if even marginal node lifts vanish under family stratification + FDR, the AST roadmap is academic) and can prioritize *which* nodes/combos to test interventionally if signal exists. (b) **#TRS-A (interventional, paired McNemar authoring-vs-template)** — the gold-standard causal test, paid. Together they triangulate: observational screens cheaply and points the interventional run at the live hypotheses.

**Methodology.** The analysis unit is the per-(primitive × target) outcome (~1,540 cells), because per-primitive "any target breached" saturates at base rate ~0.79 and hides the variance that grammar should predict. Per-node: P(breach|node) vs baseline, odds ratio + Wilson/Wald CI, Fisher exact. Pairwise interactions: observed P(breach|A&B) vs a stated no-interaction baseline → synergy score. Controls: (1) family collinearity via Cramér's V (flags circular family-mirroring nodes — if a node is ~equivalent to a family label, any lift it shows is family lift, not grammar lift); (2) Mantel–Haenszel stratification by target model (kills the target-mix confound — some models breach far more than others, and a node that co-occurs with easy targets will look predictive even if structurally uninformative); (3) within-family lift (kills the family confound — a node must show lift within a family, not just across families); (4) Benjamini–Hochberg FDR across all ~hundreds of tests (controls for multiple-comparisons inflation when testing every node × combination pair).

**Verdict criteria:** SIGNAL iff ≥1 node has an FDR-significant lift that survives family stratification and is not flagged as circular by the collinearity check. NULL (a valid, roadmap-informative result) iff no node clears that bar after controls — which means grammar structure carries little marginal signal once family is controlled and investing in AST elaboration is deprioritized. Both directions are publishable: positive identifies which components compose; null is a cheap falsification that redirects engineering effort.

**Run command:** `python scripts/grammar_study.py`

**Relationship to #TRS-A:** if #TRS-C finds node/combination signal, the powered paired McNemar test (#TRS-A, N≥60/arm) confirms causally whether structuring a technique around those nodes raises breach rate above the freeform-authoring baseline. If #TRS-C is null, #TRS-A loses its rationale (no grammar signal to causal-test). Running #TRS-C first is the canonical cheap-screen-before-expensive-confirm pattern.

**Important confound disclosures (cannot fully remove, must report):** selection bias (which primitives exist is a function of harvest history, not a random sample of the grammar space); target-mix confound (handled by MH stratification but not fully eliminated); family collinearity (flagged but not algebraically separable when a node is essentially a synonym for a family); the v1/v2 judge grading of `breach_matrix` (breach rates may shift under v3, so findings are v1/v2-calibrated; replicating under v3 post-re-judge is a robustness check, not a blocker); the 0.79 per-primitive ceiling (motivates the per-(primitive×target) unit — see methodology).

**Cost:** $0. No generation, no paid API calls. Analysis runs over the existing `breach_results` / `attack_primitives` tables and whatever grammar-labeling script labels the corpus offline.

## 🔴 #TRS-A — Powered breach-measurement gate: resolve the authoring-vs-template gap (OPEN, blocking scale investment)

**Status: open — the prerequisite before investing in generative grammar, AST composition, or synthetic technique growth.** The §10.9 slot-fill A/B (`scripts/grammar_efficacy_ab.py`; writeup `docs/scheduler_allocation_study.md §7`) is statistically underpowered: N≈16/arm, arms swapped order across runs, and variance crossover prevents any conclusion about whether structured composition raises breach rate. But beneath that underpowered A/B sits a striking unattributed effect: **deterministic templates breached ~0.25 vs freeform model authoring ~0.44** (both at 1.00 validity, zero orchestration failures). The gap is large and real but its cause is unknown — it could be template rigidity, prompt framing, coverage, or run-to-run variance. The bottleneck is **statistical power in breach measurement**, not generative expressiveness. Until a properly powered experiment (paired McNemar design, N≥60/arm, same-goal pairing, pre-registered stopping rule) is run against a fixed target with a fixed corpus, the question "does structured composition predict breach?" is unanswered — and spending engineering time on grammar elaboration or AST synthesis is premature.

**Why this gates everything downstream:** the Technique Retrieval System (`scripts/retrieval_eval.py`, Weeks 6-7) scales the candidate set from tens to hundreds or thousands, making technique evaluation economically feasible. But retrieval scales the *input* to breach measurement; it does not resolve what breach measurement can distinguish. More techniques you can't tell apart isn't progress. Closing the authoring-efficacy question is the prerequisite for knowing whether composition/generation work buys anything, and for setting the K-budget in retrieval. This is the highest-leverage experiment that is both open and affordable — a powered paired run against Claude Haiku on AdvBench goals costs roughly the same as the K-saturation probe.

**Suggested design:** 60–80 AdvBench goals, same-goal pairing (each goal gets both template and freeform authoring, independently judged), Claude Haiku fixed target, judge v3, pre-registered primary metric = McNemar p-value on per-goal breach indicator, secondary = cost-per-breach. Run once; read the sign and effect size before any grammar/synthesis build.
