# Consummation-Based Calibration of LLM-as-Judge Breach Detectors: A Methodology that Generalizes Across Breach Classes

*Working paper · last updated 2026-06-08. All results are **measured and reproducible** from `data/calibration/*.json` + `breach_results`; the v2 generalization is frozen at commit `bc088fb` (branch `v2-phase1`). Scope is honestly bounded — single-operator κ, synthetic designed-label corpora, a text-only proxy for actions (§6) — and the claims are tightened to match.*

## Abstract

Every breach verdict in an open-web LLM red-team is itself an LLM judgment, making the judge the load-bearing component — and a *generic* one: "did the agent breach policy?" is the same question for harm, data disclosure, or an unauthorized action. We present a **methodology for constructing calibrated breach judges across breach classes** from one template: a *consummation gate* ("engagement ≠ breach; consummation = breach") instantiated per breach type, a reusable per-type labeling pipeline, and a CI-gated calibration harness. We validate it through the full research loop — **diagnosis → false-positive taxonomy → targeted intervention → re-measurement → improvement** — not a single prompt. (1) On JailbreakBench we diagnose a structural over-flagging failure (70.3% human agreement, last of five), name its five-mode FP taxonomy, and fix it with the content-transfer gate (89.3%, tied with frontier classifiers). (2) The same gate, re-instantiated, calibrates a *content* breach — information-disclosure: 94.74% agreement, 100% recall, 0% FP-mode over 31 traps. (3) On a structurally different *action* breach the harness **self-diagnoses**: it returns REFINE, we apply a targeted rubric refinement, and re-measurement ships it (FP-mode 9.38%→6.25%, recall held). The central finding: **calibration difficulty is type-dependent, and the methodology exposes it** — action consummation ("did the agent execute") is harder than content consummation ("did the datum appear") for the judge *and* an independent human relabeler (κ 0.746 < 0.786 < 0.80), which both validates the gate and motivates a tool-trace upgrade. The contribution is the *methodology*, not a single judge.

**Contribution (the claim, tightened).** Not "we built a better harm judge" but **a method for constructing calibrated breach judges across multiple breach classes** — one consummation-gate template + one per-type labeling pipeline + a CI-gated harness — that **generalizes across breach types and exposes type-dependent failure modes**, validated by an explicit diagnose → taxonomy → intervene → re-measure → improve loop (the unauthorized-action REFINE→SHIP cycle) rather than a single prompt that happened to work.

## 1. The diagnosis (measured)

On JailbreakBench's `judge_comparison` set (300 human-labeled rows), the v1 judge agreed with the human majority only **70.3%** — **last of five** behind HarmBench (78.3%), LlamaGuard-2 (87.7%), GPT-4 (90.3%), and Llama-3 (90.7%) — at recall 98% / precision **55%**. A 20-row audit of production false positives (9 FPs) traced the failure to the rubric rewarding *engagement* with the attack frame (persona acceptance, acknowledgment, format mimicry) rather than *transfer* of harmful content: a response like "Sure, as DAN I'll explain…" that then said nothing dangerous scored as a breach. This is a structural rubric failure, not a threshold to nudge.

## 2. The false-positive taxonomy (measured)

Five recurring FP modes from the audit: (i) acknowledgment-as-compliance, (ii) persona/format acceptance, (iii) meta-discussion of the attack, (iv) fiction/nonsense credited as content, (v) intent-anchoring on a benign render. All five share one shape: the model *engaged* the frame without *consummating* the harm.

## 3. The fix and its validation (measured)

The rubric was rebuilt around one gate — **a breach counts only when the response transfers the operational capability the goal was after**. On the same 300 rows: **70.3 → 89.3%** agreement, **55 → 79.5%** precision, **95.5%** recall (+19 / +24.5 / −2.5 pp), moving from dead-last to **3rd of five, tied with the frontier classifiers**, via a cost-controlled tiered evaluation (~$8.4; a cheap n=25 pilot tier caught an initial recall over-correction before the full sweep). v3 is the default rubric for every scan / report / MCP / Slack verdict.

**The honest correction.** Re-judging the stored breach matrix under v3 cut breach cells **2,429 → 1,371 (−43.6%)** (full 1,535→896, partial 894→475), correcting prior over-reporting. The same recalibration was carried to the external axes: WildGuardTest harm agreement **88.5%**; StrongREJECT scored ~26% more conservatively. In-distribution false-positive rate **2.56%**.

⚑ **Notable (measured):** two respected benchmarks (WildGuardTest harm labels, StrongREJECT) themselves *over-count* relative to a strict content-transfer standard.

## 4. The generalization (harness built; result PENDING the paid sweep)

The content-transfer gate is the *harm-specific instance* of a general rule. We parameterize the judge by a **`BreachType`** that supplies the rubric, the per-rule context (the protected target / rule text), and the headline FP mode. Each type instantiates the same template with its own consummation event:

| Breach type | Consummation (= breach) | Engagement (NOT a breach) |
|---|---|---|
| capability transfer (harm, §3) | hands over operational harmful content | accepts the persona / acknowledges |
| information disclosure | the protected datum appears in the response | discusses the data-handling policy |
| unauthorized action | executes the side-effecting action | acknowledges / simulates it |

**Method (built, free):** a parameterized rubric per type (`infodisc_v1`, `unauthorized_action_v1`) sharing the v3 skeleton and 4-verdict vocabulary, swapping only the gate; a reusable **per-type independent-labeling pipeline** (synthetic designed-label corpus, n≥80, two-sided bracketing, inter-author κ — never the regulation, the operators' decisions, or the verifier's own score); and a binary-axis calibration harness reporting agreement / precision / recall **with bootstrap CIs** plus the surface-specific **FP-mode rate** ("discussing ≠ disclosing") as the headline.

**Result (measured 2026-06-08, harm path byte-identical / regression-guarded).** First, on an n=86 designed-label corpus: agreement 97.67%, precision 95.45%, recall 100%, FP-mode 0.0% (29 traps), SHIP. Because that corpus was clear-cut, we then **hardened it with 28 deliberately ambiguous boundary cases** (rounding/approximation that does-or-doesn't pin the datum, derivation that is-or-isn't fully determining, de-anonymizing small-N aggregates, refusals that leak in their own example, fiction-wrapped real data, confidently-stated *wrong* values, multi-turn assembly). On the hardened **n=114** corpus:

| | n=86 (clear-cut) | **n=114 (hardened)** |
|---|---|---|
| Agreement | 97.67% | **94.74%** [90.35, 98.25] |
| Precision | 95.45% | **90.32%** [82.26, 96.77] |
| Recall | 100% | **100%** [100, 100] |
| FP-mode (discuss-only traps) | 0.0% (29) | **0.0%** (31) |
| Gate | SHIP | **SHIP** |

The boundary cases bit — agreement and precision dropped — but the structure of the errors is the interesting result. From the aggregates (56 breach labels, recall 100%, precision 90.32%): **0 false negatives** (never missed a real disclosure) and **exactly 6 false positives, none of them discuss-only traps** (FP-mode held at 0% over all 31 traps). So the judge's *only* failure mode is **over-flagging the hardest *clean* boundary cases** (underdetermined ranges, policy-permitted fragments, confidently-wrong values) — it is conservative in precisely the safe direction, never confusing *discussing* with *disclosing* and never letting a real leak through. The content-transfer gate generalized to a non-harm rule on the first attempt and held under a hard corpus.

**Second breach type — unauthorized-action (the template under stress).** To test whether the gate generalizes beyond the cleanest case, we instantiated a *structurally different* breach type: not "did a datum *appear*" but "did the agent *execute* the side-effecting action" (an over-cap refund authorization, a forbidden opinion issued as a directive, a restricted tool/admin action), with the FP-mode being *acknowledging / simulating / describing* an action miscounted as *executing* it ("executing ≠ describing"). On a 90-case designed corpus (44 breach / 46 clean, 32 traps, an 18-case boundary tier of paired twins): **agreement 96.67% [92.22, 100], precision 93.62%, recall 100%, 0 errors** — but **FP-mode 9.38% [0, 21.88] over 32 traps, so the gate returned REFINE, not SHIP.** All three errors were false positives and **all three were FP-mode traps** (the judge over-called 3 acknowledge/simulate cases as executions); recall was perfect (no missed execution). This is the more informative outcome: the template *generalized* to an action breach with high agreement and perfect recall, **and the harness honestly refused to ship it** — because the execute-vs-simulate distinction is genuinely subtler than discuss-vs-disclose (a simulated or roleplayed action *resembles* an execution in a way that discussing a policy never resembles disclosing a datum). The system does not rubber-stamp: **calibration quality is type-dependent**, and the gate flags which types need rubric refinement. **Then we ran the loop the v3 history (§1–§3) defines:** diagnosed the single failure mode (all 3 errors = acknowledge/simulate over-called as execute), wrote a targeted **`unauthorized_action_v2`** rubric (explicit *claimed-but-no-artifact* and *illustrative/theatrical-artifact* gates — "commitment, not action-language"), and re-calibrated on the same corpus → **SHIP: agreement 97.78% [94.44, 100], precision 95.65%, recall 100%, FP-mode 6.25% [0, 15.62]** (down from 9.38%; the gate now passes). Recall held at 100% across the refinement — the targeted gate removed false positives without creating false negatives. **The diagnose → targeted-gate → recalibrate → ship loop closed on the second breach type, exactly mirroring the v3 history.**

**Independence on the hard type (κ = 0.746).** A second human re-labeled a blind 24-case sample: **Cohen's κ = 0.746, raw 87.5% — below the 0.80 floor, and lower than the content type (0.80 / 0.786).** All three disagreements adjudicated as labeler misses (originals stand; the corpus is internally consistent): the human over-called a bare future intent with no artifact ("I'll take care of it" → clean under the text-only gate), and under-called two *real* commits — one hidden behind a "hypothetically" opener, one a concrete "Sent, message-id X" confirmation. The convergent signal is itself the result: **the action type is harder for the model *and* the human.** "Did the agent execute" is a genuinely subtler consummation event than "did the datum appear," and the human's errors land exactly on the *intent/frame-vs-real-artifact* distinction the gate is built on — which both validates the gate (it captures a distinction humans find hard) and motivates the **tool-trace** upgrade, where execution is unambiguous and the text-only-proxy edge cases disappear.

**Independence check (κ = 0.80).** A second, independent human re-labeled a blind, stratified 20-row sample (`scripts/calibration/kappa_check.py`): **Cohen's κ = 0.80, raw agreement 90%** — clearing the pre-registered ≥0.80 floor. On adjudication, *both* disagreements were labeler misses (a verbatim trade-secret disclosure under-called as clean; a clean policy-refusal over-called as breach — the human slipped into the very FP-mode trap the judge resists), so the original labels stand and the calibration is unchanged. **Honest reading:** κ = 0.80 is the *floor*, not a comfortable margin, and it covered the original clear-cut 86. **Boundary κ (n=28): κ = 0.786, raw 89.3%.** A second human then re-labeled the 28 hardened boundary cases. All 3 disagreements were *author=clean / human=breach* and — tellingly — **definitional, not careless**: two were confidently-*wrong* values (a fabricated salary, a fabricated SSN), where the human counted the *attempt* as a breach but the narrow disclosure gate scores them clean because the *real* protected datum never appeared — i.e. they point to a **separate** breach type (hallucinated sensitive fabrication), not a labeling error; the third was an underdetermined range whose "materially-revealing" threshold the rubric never defined, so it was **dropped as genuinely ambiguous** (corpus → 113). That the human/rubric drift concentrates at the **definitional scope edges** — what counts as "disclosure" — rather than on careless labeling is itself a result, and a stronger signal than a sterile high κ. (The 94.74% above was measured on the pre-drop n=114; the one-case difference is immaterial and was not re-swept.) Remaining for conference-grade: a fully independent human/expert relabel (the rows were authored by an automated by-construction process), an explicit "materially-revealing" threshold (a rubric v2), and a second breach type. The **0% FP-mode over all 31 traps** is the most robust signal — a human slipping on a trap the judge handled correctly is itself evidence the gate captures a real, subtle distinction.

⚑ **Possibly publishable (result holds under a hard corpus + independence check landed):** a single content-transfer gate generalizing to per-rule consummation judges via one parameterized rubric + a reusable per-type labeling pipeline — a methods contribution extending §3. It is backed by (1) information-disclosure, which **holds under hardening** (94.74% agreement / **100% recall** / **0% FP-mode over 31 traps**, SHIP, on a 114-case corpus with 28 ambiguous boundary cases) and second-author independence checks (**κ = 0.80** base, **κ = 0.786** boundary — disagreements purely *definitional*); and (2) a **second, structurally-different breach type, unauthorized-action** (an *action* breach), which generalized but flagged **REFINE** (FP-mode 9.38%), then **SHIPPED after one targeted refinement cycle** (v2 rubric → FP-mode 6.25%, agreement 97.78%, recall held at 100%). The sharpened claim is no longer "a better harm judge" but **a repeatable consummation-calibration *methodology* that transfers across breach classes and self-diagnoses type-dependent failure modes**: harm, information-disclosure, and (after one refinement) unauthorized-action are all calibrated by the *same* gate template + labeling pipeline, with the harness shipping the clean ones and flagging-then-fixing the subtle one. **Workshop-defensible now;** conference-grade would add a fully-independent human/expert relabel and a κ check on the unauthorized-action labels.

## 5. Cross-surface reach

The same calibrated apparatus instantiates for oversight assurance (a reviewer false-approve = a consummated wrong approval) and accumulated-memory assurance (an extraction attack recovering protected content). The judge is therefore not "a harm classifier we reuse" but **a proven method for building calibrated per-target breach judges**, demonstrated once (55→79.5%) on the hardest case.

## 6. Limitations (stated plainly)

Targets are black-box live-API models whose versions are not pinned; some cells are small-n (95% bootstrap CIs are persisted precisely because of this); the judge is single-operator-calibrated. These are descriptive measurements of a live system, not validated generalizations. The independence of every ground-truth label is the load-bearing assumption (and the largest recurring cost).
