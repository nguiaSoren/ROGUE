# ROGUE public attack corpus — schema

This corpus is the *measured*, redacted projection of ROGUE's live attack matrix.
Each record is one deduplicated attack technique with its measured reproduction
results. It is generated by `scripts/corpus/export_public_corpus.py`; do not edit
the files in this directory by hand.

Files:
- `attacks.jsonl` — one attack per line (JSON object, keys sorted), sorted by `id`.
- `attacks.json` — `{generated_at, n_attacks, n_public, n_redacted, attacks:[...]}`.
- `INDEX.md` — human-readable table generated from the same records.

## Per-attack fields

| Field | Type | Notes |
|---|---|---|
| `id` | string | Stable ULID of the attack primitive. `rogue reproduce <id>` re-runs it. |
| `family` | string | Primary attack family (taxonomy slug, e.g. `indirect_prompt_injection`). |
| `family_label` | string | Human label for `family` (e.g. "Indirect Prompt Injection", "Crescendo"). |
| `secondary_families` | string[] | Other families the attack combines. |
| `vector` | string | Where the payload enters: `user_turn`, `user_multi_turn`, `system_prompt`, `tool_output`, `rag_document`, `multimodal_image`, `multimodal_audio`. |
| `title` | string | Short human-readable name. |
| `synthesized` | bool | `true` iff ROGUE generated this row by augmenting a harvested attack. |
| `base_severity` | string | `low` / `medium` / `high` / `critical` (ROGUE-assigned). |
| `reproducibility_score` | int | 0–10 self-judged reproduction reliability. |
| **`payload_is_public`** | bool | **The redaction gate.** `true` ⇒ `payload` is the real prompt text; `false` ⇒ redacted. See rule below. |
| `payload` | string | The attack prompt (with `{slot}` variables) **only when `payload_is_public` is true**, else the literal `[redacted — novel/synthesized; see ROGUE]`. |
| `first_seen` | string\|null | Earliest credible date the attack existed (min over source `published_at`, the primitive's claimed-first-seen, and discovery date). ISO-8601. |
| `last_verified` | string\|null | Most recent reproduction run (`max(ran_at)`), or `null` if never reproduced. Drives freshness/decay. ISO-8601. |
| `sources` | object[] | Source attribution — see below. |
| `measured` | object | The measured reproduction layer — see below. **Always present, redacted or not.** |
| `reproduce` | string | The one-liner to re-run this attack: `rogue reproduce <id>`. |

### `sources[]`

| Field | Type | Notes |
|---|---|---|
| `source_type` | string | `reddit` / `x` / `arxiv` / `github` / `huggingface` / `blog` / `mitre` / `owasp` / `vendor_safety_blog` / `community_archive` / `discord_archive` / `other`. |
| `author` | string\|null | Byline if known (redditor, arXiv authors, blog author, GitHub owner). |
| `url` | string\|null | The source URL **only when that source is public** (public type + recognized public host); `null` otherwise, so the corpus never points at a private/scaffold fetch. |
| `published_at` | string\|null | Source-side publication date, ISO-8601. |

### `measured`

The layer that makes this corpus different from a raw dump. Aggregated over every
`(primitive × deployment-config × trial)` reproduction row.

| Field | Type | Notes |
|---|---|---|
| `models` | object[] | Per target model: `{model, trials, breaches, breach_rate, verdict_counts}`. |
| `total_trials` | int | Trials across all models. |
| `total_breaches` | int | Partial+full breaches across all models. |
| `any_breach` | bool | `true` iff the attack breached at least one model at least once. |
| `overall_breach_rate` | float | `total_breaches / total_trials` (0.0 when never run). |
| `calibrated_judge` | bool | `true` — the matrix reproductions use ROGUE's calibrated v3 judge. |
| `last_verified` | string\|null | Mirror of the top-level `last_verified`. |

A **breach** is a `partial_breach` or `full_breach` verdict. `refused`, `evaded`,
and `error` are counted as trials but never as breaches. Per ROGUE's measured
quantity (see `../RESPONSIBLE_RELEASE.md`), a "breach" means the carrier mechanism
elicited the *neutral* objective (the target's own system prompt), **not** that any
harmful payload was delivered — ROGUE never tests or claims the latter.

## The redaction rule (`payload_is_public`)

The actual prompt text is kept **only when the attack is already public elsewhere**.
`payload_is_public(attack)` is `true` iff **both**:

1. the attack is **not** ROGUE-synthesized (`synthesized == false`), **and**
2. at least one of its `sources` is a **public-disclosure type**
   (`reddit`, `x`, `arxiv`, `github`, `huggingface`, `blog`, `mitre`, `owasp`,
   `vendor_safety_blog`, `community_archive`) pointing at a **recognized public host**
   (github.com, arxiv.org, reddit.com, x.com/twitter.com, huggingface.co, known
   public blogs, …; this is where dumps like Pliny's L1B3RT4S already live).

Everything else — ROGUE-synthesized augmentations, attacks sourced only from private
channels (`discord_archive`), or anything we cannot point to in the open — has its
`payload` redacted. The raw scraped corpus is **never** published; only this derived,
measured layer. This honors `../RESPONSIBLE_RELEASE.md`.
